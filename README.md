# dio-github-markdown

## Desafio de Projeto: Dominando Git e GitHub
Repositório criado para o desafio de projeto da **DIO**, focado em versionamento de código, fluxo de trabalho colaborativo e documentaçãp técnica em Markdown.

## 🛠️ Tecnologias Utilizadas
* **Sistema Operacional:** Ubuntu 24.04 LTS (Linux 🐧)
* **Hardware:** T495
* **Ferramentas:** Git, Terminal Bash, GitHub Web Editor
* **Linguagem de Marcação:** Markdown

## 🚀 O que aprendi nesse desafio
1. Configuração de ambiente Linux para desenvolvimento.
2. Autenticação segura via **Chave SSH**.
3. Fluxo de trabalho colaborativo (Branches e Pull Request).
4. Estrutura de documentos técnicos com Markdown.

## 🤝 Checklist de Colaboração
Acompanhamento das tarefas entre colaboradores:

- [X] Criar repositório e configurar o README inicial.
- [X] Configurar chaves SSh no notebook.
- [X] Realizar o `git clone` em um HD Externo.
- [ ] Adicionar Colaborador(a) nas configurações do GitHub.
- [ ] Criar uma nova branch para alterações: `git checkout -b feature/nome` 
- [ ] Abrir uma **Issues** para discutir uma melhoria
- [ ] Realizar um **Pull Request (PR)** com as mudanças

## 📁 Como Contribuir
1. Faça um **Fork** do projeto.
2. Crie uma nova branch para sua alteração:
```bash
   git checkout -b feature/minha-feature.
 ```
3. Salve suas alterações e faça um **commit**
```git commit -m "feat: Adicionando nova funcionalidade"
```
4. Envie para seu repositório:
```git push origin feature/minha-feature```
5. Abra um **Pull Request**
